# AI Signal 视觉保真记录

视觉基准：已确认的 B「杂志式精选卡片」布局与 A「有图用图，缺图补来源配色」封面策略。

| 检查项 | 基准 | 最终实现 | 处理结果 |
| --- | --- | --- | --- |
| 首屏信息 | 品牌、日期、更新状态、搜索与视图切换 | 保持同一信息层级，没有营销式 Hero | 一致 |
| 编辑式网格 | 一张头条配两张纵向卡片，随后宽窄交错 | 桌面三列网格，feature / standard / wide 三种卡片 | 一致 |
| 封面策略 | 原文图片优先，无图时使用来源色 | 抓取正文与 Open Graph 图片，缺图生成稳定的双来源色渐变 | 一致 |
| 字体与材质 | 苹果系统字体、轻边框、柔和阴影 | 使用系统字体栈；浅色与深色均采用低对比边框和分层材质 | 一致 |
| 摘要阅读 | 先站内查看，再进入原文 | 桌面居中详情层，手机全屏详情层，底部保留阅读原文操作 | 一致 |
| 响应式 | 手机单列、触控友好 | 390px 下无横向溢出，主要控件与图标触控区域不小于 44px | 一致 |
| 深色模式 | 需支持黑夜阅读 | 提供跟随系统、浅色、深色三态，并持久保存 | 一致 |
| 品牌图形 | 原计划尝试生成独立视觉素材 | 两次图像生成均遇到网络错误，改用代码绘制的信号波纹标记 | 可接受差异；避免阻塞核心体验 |
